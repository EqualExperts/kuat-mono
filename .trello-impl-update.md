## Implementation update (18 May 2026)

### Architecture
- **Config-driven catalog** — products pass `appSwitcher={{ apps, loading?, empty?, … }}` on `KuatHeader`. Kuat does not fetch apps.
- **Shared logic** — `kuat-header.logic.ts` in React and Vue (`shouldShowAppSwitcher`, label/empty resolution, lockup + account helpers).
- **Desktop** — waffle `IconButton` + dropdown panel (title + optional description + icon per app).
- **Mobile** — built-in sheet with drill-in: main → **Equal Experts apps** / **Account**; focus trap, `inert` inactive panels, Esc closes, Back pops stack.
- **Styling** — Kuat CSS in `kuat-header.css`; React/Vue parity.

### New features
- `appSwitcher` prop: `apps`, `loading`, `empty` (`hide` | `message`), `emptyMessage`, `linkTarget`, `labels`, `onOpen`, `onSelect`
- Structured `account` prop (desktop dropdown + optional `account.mobile` tier)
- `lockup={{ variant: "default" | "demo" }}` for built-in EE logos
- `IconButton` primitive + subpath (`@equal-experts/kuat-react/icon-button`, `@equal-experts/kuat-vue/icon-button`)

### Changes to existing behaviour
- Header composable from props (nav + account + app switcher) as well as legacy slots
- Built-in mobile sheet when structured menus / switcher present (unless custom `mobileMenu` overrides)
- Legacy escape hatches: `accountMarkup` (React), `#account-markup` / `#actions` (Vue)

### Breaking changes (0.12 → 0.13)
1. **Built-in logo is opt-in** — pass `lockup={{ variant: "default" }}` (or `"demo"`). No `lockup` + no custom `logo` = title-only header.
2. Prefer structured `account` over ad-hoc markup for dropdown + mobile account tier.
3. `KuatHeaderActionItem` deprecated → `KuatHeaderAccountItem` with `href`.
4. Vue: prefer `#account-markup` over `#actions`.
5. Omit `appSwitcher` to hide switcher (non-breaking for new adopters).
