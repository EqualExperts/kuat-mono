# Equal Experts Colour Palettes

A guide to using Equal Experts brand colours for digital design and development. This document helps ensure consistent, accessible, and brand-compliant colour usage across all digital products.

## Primary Palette

The primary palette consists of core brand colours that represent Equal Experts. These should be used as the foundation for all brand materials.

### Primary Colours

#### Equal Experts Blue
- **Hex:** `#1795D4`
- **RGB:** 23, 149, 212
- **CMYK:** 89, 30, 0, 17
- **Usage:** Primary brand colour. Use for primary actions, key brand elements, and main visual identity.

#### Tech Blue
- **Hex:** `#22567C`
- **RGB:** 34, 86, 124
- **CMYK:** 73, 31, 0, 51
- **Usage:** Secondary primary colour. Use for headings, important text, and secondary brand elements.

### Accent Colours

#### Transform Teal
- **Hex:** `#269C9E`
- **RGB:** 38, 156, 158
- **CMYK:** 76, 1, 0, 38
- **Usage:** Accent colour for highlights, call-to-action variations, and visual interest.

#### Equal Ember
- **Hex:** `#F07C00`
- **RGB:** 240, 124, 0
- **CMYK:** 0, 48, 100, 6
- **Usage:** Accent colour for alerts, warnings, and attention-grabbing elements.

### Neutral Colours

#### Dark Data
- **Hex:** `#212526`
- **RGB:** 33, 37, 38
- **CMYK:** 13, 3, 0, 85
- **Usage:** Primary text colour, dark backgrounds, and high-contrast elements.

#### The Cloud
- **Hex:** `#F5F5F5`
- **RGB:** 245, 245, 245
- **CMYK:** 0, 0, 0, 4
- **Usage:** Light backgrounds, subtle sections, and content areas.

#### Byte White
- **Hex:** `#FFFFFF`
- **RGB:** 255, 255, 255
- **CMYK:** 0, 0, 0, 0
- **Usage:** Page backgrounds, card backgrounds, and text on dark backgrounds.

## Usage Guidelines

### When to Use Primary vs Accent Colours

- **Primary Colours (Equal Experts Blue, Tech Blue):** Use for main brand elements, primary actions, navigation, and key content areas. These should be the dominant colours in your design.
- **Accent Colours (Transform Teal, Equal Ember):** Use sparingly for highlights, secondary actions, and visual interest. They should complement, not compete with, primary colours.

### Accessibility Requirements

Equal Experts adheres to **WCAG AA accessibility standards** for colour contrast. All colour combinations must meet these requirements:

- **Normal text (14px and below):** Minimum contrast ratio of **4.5:1**
- **Large text (18px+ or 14px+ bold):** Minimum contrast ratio of **3:1**

## Do's and Don'ts

### ✅ Do's

1. **Use approved accessible combinations**
   - Always use the tested colour combinations listed above for text on backgrounds
   - Example: ✅ Equal Experts Blue text on The Cloud background

2. **Maintain brand consistency**
   - Use Equal Experts Blue as the primary brand colour
   - Use Tech Blue for secondary elements and headings
   - Keep accent colours (Transform Teal, Equal Ember) for highlights only

3. **Ensure sufficient contrast**
   - Test all colour combinations meet WCAG AA standards
   - Use Dark Data for primary text on light backgrounds
   - Use Byte White for text on dark/coloured backgrounds

4. **Use neutral colours appropriately**
   - Use The Cloud for subtle backgrounds and sections
   - Use Byte White for clean, minimal backgrounds
   - Use Dark Data for high-contrast text

5. **Apply colours consistently across states**
   - Maintain colour relationships in hover, active, and disabled states
   - Use consistent opacity levels for disabled states

### ❌ Don'ts

1. **Don't use unapproved colour combinations**
   - ❌ Equal Experts Blue text on Byte White background (not in approved list)
   - ❌ Transform Teal text on The Cloud background for normal text (only approved for large text)
   - ❌ Equal Ember text on Byte White background (not in approved list)

2. **Don't create new colour variations**
   - ❌ Don't modify brand colours (e.g., don't use `#1795D5` instead of `#1795D4`)
   - ❌ Don't create custom tints or shades without brand approval
   - ❌ Don't use colours outside the defined palette without consultation

3. **Don't ignore accessibility**
   - ❌ Don't use low-contrast combinations (e.g., light grey text on white)
   - ❌ Don't rely solely on colour to convey information (use icons, text, patterns)
   - ❌ Don't use colour combinations that don't meet WCAG AA standards

4. **Don't overuse accent colours**
   - ❌ Don't make Transform Teal or Equal Ember the dominant colour
   - ❌ Don't use multiple accent colours in the same component
   - ❌ Don't use accent colours for primary actions (use Equal Experts Blue)

5. **Don't use placeholder colours in production**
   - ❌ Don't deploy UI with placeholder colour values
   - ❌ Always consult brand team for undefined colours (error, success, light grey)
   - ❌ Don't assume colour values for undefined UI elements

### Examples

#### ✅ Good: Accessible Button
```
Primary Button:
- Background: Equal Experts Blue (#1795D4)
- Text: Byte White (#FFFFFF)
- Contrast: 4.8:1 (meets WCAG AA)
```

#### ❌ Bad: Inaccessible Button
```
Button:
- Background: Equal Experts Blue (#1795D4)

- Contrast: 1.2:1 (fails WCAG AA)
```

- Normal text combination not in approved list, may not meet contrast requirements
